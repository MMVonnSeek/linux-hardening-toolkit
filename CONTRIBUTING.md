# Guia de Contribuição

Contribuições são bem-vindas. Para novos controles de auditoria, cada contribuição precisa de:

- Função de verificação em `audit.sh`
- Função de correção em `hardening.sh`
- Documentação em `docs/`
- Referência ao controle CIS Benchmark correspondente

Abra uma issue antes de PRs grandes para alinhar a direção.

Veja os padrões completos em [linux-security-guide/CONTRIBUTING.md](https://github.com/MMVonnSeek/linux-security-guide/blob/main/CONTRIBUTING.md).
