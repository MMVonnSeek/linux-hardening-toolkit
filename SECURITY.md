# Política de Segurança

## Escopo

Este repositório contém scripts Bash e Python para auditoria e hardening de servidores Linux. Vulnerabilidades relevantes para reporte incluem:

- Scripts que causem dano não intencional ao sistema quando executados conforme documentado
- Lógica de verificação incorreta que produza falsos negativos em controles críticos
- Vulnerabilidades de injeção de comando nos scripts de auditoria ou hardening
- Escalação de privilégios introduzida pelos próprios scripts de correção

## Como reportar

**Não abra uma issue pública para reportar vulnerabilidades.**

Entre em contato diretamente via LinkedIn:
[linkedin.com/in/max-muller-685705248](https://www.linkedin.com/in/max-muller-685705248/)

Inclua na mensagem:

1. Descrição do problema
2. Script e linha afetada
3. Condição necessária para reproduzir
4. Impacto potencial
5. Sugestão de correção, se possível

## Tempo de resposta

Confirmação em até **5 dias úteis**. Correção em até **15 dias úteis** após confirmação.

## Versões suportadas

Apenas a versão mais recente (`main`) recebe correções.
